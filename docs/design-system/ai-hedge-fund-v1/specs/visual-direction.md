# Visual Direction

## Design Intent

Build a high-signal, precision-first interface language for AI-driven trading operations.

## Aesthetic Signature

- Tone: Precision-industrial.
- Character: Serious, data-first, operator-focused.
- Memory hook: Electric blue action rails over restrained neutral surfaces.
- Mode policy: Dark mode only.

## Typography

- Primary family: Geist (fallback to system sans).
- Data/metrics family: Geist Mono or IBM Plex Mono for numeric surfaces.
- Hierarchy rule: Keep headings compact and confident; keep data labels low-noise.

## Color Strategy

- Dominant neutrals for structure and readability.
- Action primary: `#0B5FFF`.
- Action accent: `#00B8D9`.
- Status gain/loss/warn: `#11A36A`, `#D9485F`, `#D99A2B`.

## Motion

- Use brief, meaningful transitions (120ms-180ms) for state changes.
- Reserve stronger motion only for workflow transitions (run backtest, publish flow, node execution state).

## Spatial System

- Base spacing rhythm: 4px grid.
- Use clear panel layering to separate control, context, and output zones.
- Keep controls dense but avoid cramped labels in risk-sensitive views.

## Component Character

- Buttons: high-contrast primary with visible focus ring.
- Cards/panels: subtle border + restrained elevation.
- Tabs and node controls: semantic state colors over decorative styling.
