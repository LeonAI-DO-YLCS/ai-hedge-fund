# Planning Documents

This folder is the detector-local planning package for the next V75 strategy-research phase. It documents why the detector should evolve beyond broad inefficiency screening, what the next research scope should include, and how that future work should be sequenced without changing the main project runtime.

## Repository Boundary

- These planning documents apply only to `Randomness and Inefficiencies Detector/`.
- They guide future detector work and do not redefine the architecture of the main project.
- Any future integration idea must be treated as separate scope from this planning package.

## Documents in This Package

- `README.md`: planning index and navigation guide for this package.
- `v75-strategy-research-prd.md`: primary decision document covering the problem, vision, goals, non-goals, scope, risks, validation model, and success measures for the next detector phase.
- `v75-strategy-research-roadmap.md`: phased execution document covering delivery order, milestones, risks, and validation gates for the future strategy-research expansion.

## Recommended Reading Order

1. Start with `v75-strategy-research-prd.md` if you need to understand why the next research phase exists and what it is trying to achieve.
2. Continue with `v75-strategy-research-roadmap.md` if you need to understand how that future work should be sequenced and gated.
3. Return to this `README.md` whenever you need a quick index of the package contents.

## Who Should Read What

- Project leads and reviewers should read `v75-strategy-research-prd.md` first.
- Researchers and implementation planners should read `v75-strategy-research-roadmap.md` after the PRD.
- Contributors new to this folder should use `README.md` as the starting point.
